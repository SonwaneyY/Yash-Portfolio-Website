# My Work

This directory contains your case studies, project documentation, and portfolio pieces.

## What to Include

- **Case study PDFs** - Your completed project documentation
- **Project descriptions** - Brief overviews of your work
- **Process documentation** - How you approached each project
- **Research findings** - User research, insights, data
- **Design artifacts** - Screenshots, wireframes, prototypes (as PDFs or images)

## Organization Tips

Consider organizing by:
- Project name
- Domain (product design, strategic design)
- Client/company
- Year

## Example Structure

```
my-work/
├── product-design/
│   ├── project-alpha-case-study.pdf
│   └── project-beta-overview.md
└── strategic-design/
    ├── client-x-research.pdf
    └── client-y-process.md
```

## Why This Matters

The slash commands will reference this content to:
- Understand your unique voice and style
- Learn your approach to design
- Extract terminology and concepts you use
- Maintain consistency across your portfolio
