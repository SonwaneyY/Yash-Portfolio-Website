# Drafts

This is your working area for content being refined and polished.

## Organization

Keep drafts organized by content type:

```
drafts/
├── project-titles/
│   ├── healthcare-dashboard-title.md
│   └── fintech-app-title.md
├── about-page/
│   ├── bio-v1.md
│   └── bio-v2.md
├── image-captions/
│   └── dashboard-mockup-caption.md
├── navigation/
│   └── main-nav-labels.md
└── case-studies/
    ├── project-alpha-intro.md
    └── project-beta-overview.md
```

## Workflow

1. **Create a draft** - Write your initial version
2. **Run a slash command** - Use the appropriate command to refine it
3. **Iterate** - Adjust and re-run if needed
4. **Finalize** - Move to `outputs/` when satisfied

## Tips

- Include context in your draft (e.g., what the project is about)
- Note any specific requirements (word count, must-include keywords)
- Keep multiple versions to see evolution
- Add comments or notes for Claude to consider

## Draft Template

Create a file with this structure:

```markdown
# [Content Type] - [Project/Section Name]

## Context
<!-- What is this for? Where will it appear? -->

## Requirements
- Word count: ~X words
- Must include: [keywords, concepts]
- Tone: [professional, casual, etc.]

## Draft

[Your content here]

## Notes
<!-- Anything specific you want the AI to consider -->
```
